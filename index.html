🧩 Projekt-Blueprint — Übersicht

Statisches HTML-Mehrseiten-Template · Tabs, Dark/Light-Mode, einklappbare Sektionen, Karten-Grid · Jede Karte führt zu einer eigenen Detailseite
🎯
ZielgruppeHier stehen die wichtigsten Rahmenbedingungen des Projekts — wer es nutzt, was es leisten soll, was zu beachten ist.
⚙️
TechnikKein Build, kein Framework · läuft per file:// direkt im Browser · jede Seite self-contained (eigenes CSS & JS).
🎨
DesignDark Mode als Default, Light-Override per CSS-Variablen · Akzentfarben blue/teal/green/amber/coral/purple.
📐
KonventionenDetails in der CLAUDE.md dieses Ordners — Klassenkatalog, Fallstricke, Checklisten für neue Seiten.

Hintergrund-Infos zu Kategorie A

📋 Info-Karten — kurze Wissenshäppchen
💡
Erster Hinweis
Info-Karten eignen sich für gleichrangige, kurze Wissenseinheiten — Regeln, Tipps, Unterschiede. Drei bis acht Karten pro Grid.
⚠️
Zweiter Hinweis
Jede Karte: Icon, fetter Kurztitel, zwei bis vier Sätze Fließtext. Keine Links in Info-Karten — dafür gibt es die Cards auf den Detailseiten.
🔧
Dritter Hinweis
Das Grid bricht automatisch um (auto-fit, min 200px) — keine media queries pro Karte nötig.
